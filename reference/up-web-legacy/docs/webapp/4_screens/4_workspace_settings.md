# 4. Workspace Settings - ワークスペース設定・管理画面設計

## 概要

ワークスペース内の設定・管理画面。管理者のみがアクセス可能。ワークスペース設定、メンバー管理、Google Drive連携などを行う。

## 画面一覧

| 画面ID | 画面名 | パス | 説明 | アクセス権限 |
|--------|--------|------|------|-------------|
| WS-001 | ダッシュボード | `/[slug]/dashboard` | ワークスペースホーム | 全メンバー |
| WS-002 | ワークスペース設定 | `/[slug]/settings` | 基本設定 | 管理者のみ |
| WS-003 | メンバー管理 | `/[slug]/members` | メンバー招待・管理 | 管理者のみ |
| WS-004 | Google Drive連携 | `/[slug]/settings/drive` | Drive接続設定 | 管理者のみ |

## ユーザーシナリオ

### シナリオ 1: ワークスペース設定の変更

```mermaid
sequenceDiagram
    actor Admin as 管理者
    participant Sidebar as サイドバー
    participant Settings as 設定画面
    participant DB as Database

    Admin->>Sidebar: 「ワークスペース設定」クリック
    Sidebar->>Settings: 設定画面へ遷移

    Admin->>Settings: ワークスペース名を変更
    Admin->>Settings: 「変更を保存」クリック
    Settings->>DB: 更新リクエスト
    DB-->>Settings: 成功
    Settings->>Admin: 「保存しました」トースト
```

### シナリオ 2: 新しいメンバーの招待

```mermaid
sequenceDiagram
    actor Admin as 管理者
    participant Members as メンバー管理
    participant Dialog as 招待ダイアログ
    participant Email as メールシステム
    participant Invitee as 招待されたユーザー

    Admin->>Members: 「メンバーを招待」クリック
    Members->>Dialog: 招待ダイアログを開く

    Admin->>Dialog: メールアドレス入力
    Admin->>Dialog: 役割選択（メンバー）
    Admin->>Dialog: 「招待を送信」クリック

    Dialog->>Email: 招待メール送信
    Dialog->>Members: 「招待を送信しました」表示
    Note over Members: 「保留中の招待」に追加

    Email->>Invitee: 招待メール受信
    Note over Invitee: 承認後、メンバーとして参加
```

### シナリオ 3: Google Drive連携（Service Account方式）

```mermaid
sequenceDiagram
    actor Admin as 管理者
    participant Drive as Drive設定画面
    participant API as Next.js API
    participant DB as Database
    participant GDrive as Google Drive API

    Admin->>Drive: Drive設定画面へ
    Drive->>Admin: 未連携状態を表示（JSONアップロードUI）

    Admin->>Drive: Service Account JSONをアップロード
    Drive->>Drive: クライアント側バリデーション

    alt バリデーション失敗
        Drive->>Admin: エラー表示（必須フィールド不足等）
    else バリデーション成功
        Drive->>Admin: JSON検証成功、フォルダID入力フォーム表示

        Admin->>Drive: フォルダIDを入力（任意）
        Admin->>Drive: 「設定を保存」クリック

        Drive->>API: POST /api/workspace/drive
        API->>API: サーバー側バリデーション
        API->>DB: 暗号化してService Account保存
        DB-->>API: 保存成功
        API-->>Drive: 成功レスポンス（client_email）
        Drive->>Admin: 「設定が完了しました」表示

        Note over Drive: 連携済み状態に遷移
    end
```

### シナリオ 3-2: 接続テスト

```mermaid
sequenceDiagram
    actor Admin as 管理者
    participant Drive as Drive設定画面
    participant API as Next.js API
    participant DB as Database
    participant GDrive as Google Drive API

    Admin->>Drive: 「接続テスト」クリック
    Drive->>API: POST /api/workspace/drive/test
    API->>DB: Service Account取得・復号化
    DB-->>API: Service Account JSON
    API->>GDrive: 共有ドライブ一覧取得

    alt 接続成功
        GDrive-->>API: ドライブ情報返却
        API-->>Drive: 成功レスポンス + ドライブ一覧
        Drive->>Admin: 接続テスト成功ダイアログ
    else 接続失敗
        GDrive-->>API: エラー（権限不足等）
        API-->>Drive: エラーレスポンス + 原因
        Drive->>Admin: 接続テスト失敗ダイアログ + 解決方法
    end
```

### シナリオ 4: メンバーの役割変更

```mermaid
sequenceDiagram
    actor Admin as 管理者
    participant Members as メンバー管理
    participant Dialog as 役割変更ダイアログ
    participant DB as Database

    Admin->>Members: メンバー行の「役割」ドロップダウンクリック
    Members->>Admin: 役割オプション表示

    Admin->>Members: 「管理者」を選択
    Members->>Dialog: 確認ダイアログ表示

    Admin->>Dialog: 「変更する」クリック
    Dialog->>DB: 役割更新
    DB-->>Members: 成功
    Members->>Admin: 役割が更新されました
```
