# TV番組放送回データ管理ガイド

> **作成日**: 2026-02-27  
> **作成理由**: 誤った放送回データがAIに提供される事態を防ぐため

---

## 背景

2026年2月27日、13番組×5回分=65件の放送回データを検証した結果、以下の重大な誤りが発見されました：

- 放送局誤り：6番組
- 曜日・時刻誤り：4番組
- 存在しない放送回：複数
- タイトル誤り：ほぼ全件

これらの誤りは、**検証なしにデータがコミットされた**ことによるものです。

---

## 検証チェックリスト

### 基本検証（必須）

- [ ] **放送局**が正しいか
  - マツコ→TBS
  - しくじり→テレビ朝日
  - 鬼レンチャン→フジテレビ（日本テレビではない）
  - 熱狂マニアさん→TBS（日本テレビではない）

- [ ] **曜日と時刻**が正しいか
  - 公式サイトの番組表で確認
  - 特別編成時は注意

- [ ] **日付と曜日**が一致するか
  - 2026-02-24→火曜日
  - カレンダーで必ず確認

- [ ] **タイトル**が正しいか
  - 公式サイトの表記と完全一致
  - 勝手に詳細を追加しない

### 拡張検証（推奨）

- [ ] **ゲスト**が正しいか
  - 複数ソースで確認
  - 公式発表を優先

- [ ] **内容概要**が正しいか
  - 実際の放送内容と一致
  - 予告編とは区別

## よくある誤りパターン

### 1. 放送局の混同

```
❌ 千鳥の鬼レンチャン → 日本テレビ
✅ 千鳥の鬼レンチャン → フジテレビ

❌ 熱狂マニアさん！ → 日本テレビ
✅ 熱狂マニアさん！ → TBS

❌ THE神業チャレンジ → 日本テレビ
✅ THE神業チャレンジ → TBS

❌ ニカゲーム → テレビ東京
✅ ニカゲーム → テレビ朝日
```

### 2. タイトルの創作

```
❌ 「新春拡大SP 演歌の世界」（創作）
✅ 「歌舞伎女方の世界」（公式）

❌ 「今こそ食べて欲しい みかんVSりんご...」（創作）
✅ 「医師33人がジャッジ みかんVSりんご...」（公式）
```

### 3. 曜日の混同

```
❌ 千鳥の鬼レンチャン → 水曜
✅ 千鳥の鬼レンチャン → 日曜

❌ 2025-02-05（水曜）に放送
✅ 日曜番組なので水曜放送は存在しない
```

## データ作成手順

### Step 1: 公式サイトで情報収集

```bash
# 例：マツコの知らない世界
# https://www.tbs.co.jp/matsuko-sekai/archive/2026.html
# から直近5回分の情報を収集
```

収集項目：
- 放送日（YYYY-MM-DD形式）
- タイトル（公式表記のまま）
- ゲスト（公式表記のまま）
- 内容概要（公式サイトから引用）

### Step 2: 型に沿ってデータ作成

```typescript
{
  broadcastDate: "2026-02-24",  // 必須
  title: "野生ネコの世界",       // 推奨
  content: "...",                // 必須
  guests: ["秋山知伸"],          // 必須（空配列可）
}
```

### Step 3: サンプル検証

```bash
# 作成したデータの10-20%を検証
# 最低3件

# 検証方法例：
# 1. 放送局公式サイトで該当日の番組表を確認
# 2. 日付と曜日が一致するか確認
# 3. タイトルが一致するか確認
```

### Step 4: 問題があれば修正

問題発見時：
1. 誤りを記録
2. 修正
3. 再検証
4. 問題なければコミット

### Step 5: 検証記録を残す

```markdown
## 検証記録
- **検証日**: 2026-02-27
- **検証対象**: programs-detailed-data.ts
- **検証方法**: TBS公式サイト、テレビ朝日公式サイト
- **検証結果**: 5件サンプル検証、全て正確
- **備考**: 問題なし
```

## 緊急時の対応（誤り発見時）

### 即座に行うこと

1. **データの無効化**
   ```typescript
   // 誤りが見つかったため一時的に無効化
   recentEpisodes: [],
   ```

2. **検証レポート作成**
   - `docs/verification/` に保存
   - 誤りの詳細を記録

3. **ユーザーに報告**
   - 誤りの概要
   - 影響範囲
   - 修正方針

### 修正後の手順

1. 公式サイトから正確なデータを収集
2. 全件検証
3. コミット
4. 検証記録を残す
