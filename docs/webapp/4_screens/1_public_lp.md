# 1. Public (LP) - ランディングページ画面設計

## 概要

T-Agentのランディングページ。未ログインユーザーに対してサービスの価値を訴求し、会員登録・ログインへ誘導する。

## 画面一覧

| 画面ID | 画面名 | パス | 説明 |
|--------|--------|------|------|
| LP-001 | ランディングページ | `/` | メインLP |
| LP-002 | 料金プラン | `/pricing` | 料金体系 |
| LP-003 | 機能紹介 | `/features` | 詳細機能説明 |
| LP-004 | お問い合わせ | `/contact` | 問い合わせフォーム |

### コンポーネント構成

| コンポーネント | 説明 | 状態 |
|---------------|------|------|
| `Header` | ナビゲーションバー | スクロール時に固定表示 |
| `HeroSection` | メインキャッチコピー + CTA | アニメーション付き |
| `ProblemSection` | 課題提示カード x 3 | ホバーエフェクト |
| `SolutionSection` | 解決策カード x 3 | ホバーエフェクト |
| `AgentTypesSection` | エージェント紹介 x 4 | カルーセル（モバイル） |
| `WorkflowSection` | ステップ図解 | フローアニメーション |
| `CTASection` | 登録誘導 | ボタン強調 |
| `Footer` | フッターリンク | - |

## ユーザーシナリオ

### シナリオ 1: 初回訪問（検索流入）

```mermaid
sequenceDiagram
    actor User as 制作会社AD
    participant LP as ランディングページ
    participant Auth as 認証ページ

    User->>LP: 「番組制作 AI 効率化」で検索、LPに到着
    LP->>User: ヒーローセクション表示
    Note over User: "番組制作を、AIで加速する" を見る

    User->>LP: スクロールダウン
    LP->>User: 課題セクション表示
    Note over User: "自分たちの課題と同じだ" と認識

    User->>LP: 続けてスクロール
    LP->>User: 解決策・エージェント紹介表示
    Note over User: 具体的な機能を理解

    User->>LP: 「無料で始める」クリック
    LP->>Auth: 会員登録ページへ遷移
```

### シナリオ 2: 比較検討中のユーザー

```mermaid
sequenceDiagram
    actor User as ディレクター
    participant LP as ランディングページ
    participant Pricing as 料金ページ
    participant Contact as お問い合わせ

    User->>LP: 同僚から聞いて訪問
    LP->>User: LP表示

    User->>LP: ヘッダーの「料金」クリック
    LP->>Pricing: 料金ページへ遷移
    Pricing->>User: プラン比較表示
    Note over User: 自社の規模に合うプランを確認

    User->>Pricing: 「お問い合わせ」クリック
    Pricing->>Contact: お問い合わせフォームへ
    User->>Contact: 問い合わせ内容入力
    Contact->>User: 送信完了メッセージ
```

### シナリオ 3: 既存ユーザーの再訪問

```mermaid
sequenceDiagram
    actor User as 既存ユーザー
    participant LP as ランディングページ
    participant Auth as 認証ページ
    participant MyPage as マイページ

    User->>LP: ブックマークから訪問
    LP->>User: LP表示

    User->>LP: ヘッダーの「ログイン」クリック
    LP->>Auth: ログインページへ遷移

    User->>Auth: 認証情報入力
    Auth->>MyPage: ログイン成功、マイページへ
```
