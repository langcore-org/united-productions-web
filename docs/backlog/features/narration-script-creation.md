## 背景

ナレーション原稿作成機能について、Word形式への出力実装が困難であること、
中途半端な完成度では現場で使われなくなるリスクがあることから、
第1段階リリースからは外し、時間をかけてフルセットで実装することを検討する。

## 現状の課題

### 技術的課題

- 表形式のデータをWordに綺麗に変換する実装が困難
- 出力が崩れると作業が無意味になる

### ユーザビリティ課題

- 表のテキストコピペだけでは現場のワークフローに合わない
- 初回の印象が重要

## フルセット実装案

### 案A: 画面上編集 + Word出力

```
【機能構成】
1. 表形式の原稿編集画面
   - タイムコード列
   - 発言者列
   - セリフ列
   - 行の追加・削除
   - ドラッグ&ドロップで順序変更

2. AIアシスト機能
   - セリフの自動生成
   - タイムコードの自動計算
   - 修正案の提案

3. Word出力
   - 整形されたWord文書出力
   - テンプレート対応
```

### 案B: Canvas機能的アプローチ

```
【機能構成】
1. ドキュメント編集画面（Gemini Canvas風）
   - 自由形式の編集
   - AIとの対話しながら編集

2. 原稿テンプレート
   - ナレーション用フォーマット
   - カスタマイズ可能

3. エクスポート機能
   - Word形式
   - PDF形式
   - プレーンテキスト
```

## 実装工数見積もり

| 案 | 工数 | 優先度 |
|----|------|--------|
| 案A | 2〜3週間 | 高 |
| 案B | 3〜4週間 | 中 |
