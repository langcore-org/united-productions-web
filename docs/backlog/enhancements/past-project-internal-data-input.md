## 背景

新規企画立案機能において、社内情報である過去企画のインプットが検討されたが、
以下の課題により第1段階では見送り、第2段階で検討する。

## 課題

### 1. 権限管理の複雑性

```
【必要な機能】
- 誰がどの番組に関与しているかの管理
- 役職・関与度によるアクセス権限の設定
- 情報漏洩防止のための制御
```

### 2. データ整備の工数

```
【必要な作業】
- 過去企画の一覧作成
- 各企画の詳細情報整理
- 関係者情報の紐付け
```

### 3. ミス防止のための機能実装

```
【必要な機能】
- 誤って関係ない人が情報を見られないようにする制御
- 検索時のアクセス制限
- 監査ログ
```

## 実装案

### Phase 1: データベース設計（4月）

```
【スキーマ案】
- projects（企画テーブル）
  - id, name, description, genre, year, status

- project_members（企画-メンバー関連）
  - project_id, user_id, role, access_level

- access_logs（アクセスログ）
  - user_id, project_id, action, timestamp
```

### Phase 2: 権限管理機能（5月）

```
【機能】
- ロールベースアクセス制御（RBAC）
- プロジェクト単位のアクセス制限
- 管理者ダッシュボード
```

### Phase 3: データ移行（6月）

```
【作業】
- 過去企画データの移行
- 関係者情報の紐付け
- 検証
```
