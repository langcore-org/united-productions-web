### 実装方針

#### オプションA: 独立機能として実装

```
サイドバー構成：
├─ チャット
├─ 出演者リサーチ
├─ エビデンスリサーチ
├─ SNSリサーチ（YouTube/Instagram）  ← 新規
├─ 議事録作成
├─ 新企画立案
└─ 履歴
```

#### オプションB: 出演者リサーチに統合（推奨）

```
出演者リサーチ内：
- X検索（既存）
- YouTube検索（追加）
- Instagram検索（追加）
- Web検索（既存）
```

**推奨**: オプションB（出演者リサーチに統合）
- 同じ「人物」に関するリサーチを一元化
- UIがシンプルになる

## 仕様案

### 検索対象

| プラットフォーム | 検索対象 | 用途 |
|------------------|----------|------|
| YouTube | 出演動画、チャンネル情報 | 過去の出演、人気動画 |
| Instagram | 投稿内容、フォロワー数 | ファン層、トレンド |

### 出力形式

出演者リサーチの結果に以下を追加：

```markdown
### 📱 SNSアカウント（追加情報）
#### YouTube
- **チャンネル**: [チャンネル名](URL)
- **登録者数**: [数値]
- **代表作**: [動画タイトル](URL)

#### Instagram
- **アカウント**: [@username](URL)
- **フォロワー数**: [数値]
- **最近の投稿**: [投稿内容の概要]
```

## リスクと対策

| リスク | 対策 |
|--------|------|
| 費用承認が下りない | 無料APIの活用や縮小版の検討 |
| 実装に時間がかかる | Phase 1で工数見積もりを厳密に行い、無理なら縮小 |
| API制限に引っかかる | レート制限の監視、フォールバック処理の実装 |
