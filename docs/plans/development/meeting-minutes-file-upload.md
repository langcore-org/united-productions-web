> **期限**: 2026-03-12  
> **担当**: 浅野宏耀  
> **ステータス**: ✅ **実装済み（テスト待ち）**  
> **親タスク**: [実装プラン 2026年3月](../implementation-plan-2026-03.md)

---

## 目的

議事録作成機能を「VTT/TXTファイルのアップロード」形式に変更し、現場のワークフローに最適化する。

## 背景

- 当初はテキストコピペも検討していたが、ファイルアップロードが最も効率的と判断
- Zoomから出力されるVTTまたはTXTファイルをそのまま使用

## 仕様

### 対応ファイル形式

| 形式 | 拡張子 | 優先度 | 状態 |
|------|--------|--------|------|
| VTT | `.vtt` | 高 | ✅ 実装済み |
| TXT | `.txt` | 高 | ✅ 実装済み |
| DOCX | `.docx` | 中 | ✅ 実装済み |

### VTTファイル仕様

Zoomが出力するVTT形式：
```
WEBVTT

00:00:00.000 --> 00:00:05.000
発言者A: こんにちは、会議を始めます。

00:00:05.000 --> 00:00:10.000
発言者B: はい、お願いします。
```

### 処理フロー

```
1. ユーザーがファイルを選択・アップロード
2. ファイル形式を検証
3. 内容をパース（VTT→テキスト抽出）
4. AIに送信し、議事録を生成
5. 整形された議事録を表示
```

### 2. バックエンド実装

- [x] ファイル受信API (`/api/upload`)
- [x] ファイル検証（形式: VTT/TXT/DOCX、サイズ: 10MB上限）
- [x] VTTパーサー実装（`lib/parsers/vtt.ts`・`lib/upload/file-parser.ts`）
- [x] TXTパーサー実装（文字コード自動判定: UTF-8/Shift_JIS）
- [x] DOCXパーサー実装（mammoth使用）
- [x] AIへの送信処理 (`/api/meeting-notes`・`/api/llm/stream`)

### 3. プロンプト実装

- [x] 議事録生成プロンプト（`prompts/meeting-format.ts`）
- [x] テンプレート選択（会議用・面談用）
- [x] DB連携（`getPromptFromDB`）

【入力】
- 会議のテキストデータ（VTT/TXTから抽出）

【出力形式】
1. 会議概要
   - 日時
   - 参加者（抽出できれば）
   - 議題

2. 決定事項
   - 決定された内容を箇条書き

3. アクショナイテム
   - 担当者（わかれば）
   - 期限（わかれば）
   - タスク内容

4. 議論のポイント
   - 重要な議論を要約

5. 次回予定（あれば）
