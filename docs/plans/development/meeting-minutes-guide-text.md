> **優先度**: 🟡 中  
> **期限**: 2026-03-12  
> **担当**: 浅野宏耀  
> **ステータス**: 未着手  
> **親タスク**: [実装プラン 2026年3月](../implementation-plan-2026-03.md)

---

## 目的

議事録作成機能の初期画面に表示するガイド文言・ラベルを整備し、ユーザーがVTT/TXTファイルをアップロードすることを明確に案内する。

## 背景

- 議事録作成は「ファイルアップロードを基本」とする形に変更
- ZoomのVTT/TXTファイルをアップロードすることをユーザーに伝える必要がある
- 初見のユーザーでも迷わず使えるような案内が必要

## ガイド文言案

### 初期画面（ファイルアップロード前）

```
【タイトル】
議事録作成

【サブタイトル】
Zoomの録画ファイルから議事録を自動生成します

【ガイド文】
📁 アップロードするファイル
Zoomで録画した会議の「字幕ファイル（.vtt）」または「テキストファイル（.txt）」をアップロードしてください。

【手順】
1. Zoomで「レコード」をクリック
2. 会議終了後、Zoomの「レコーディング」ページを開く
3. 「字幕ファイル」（.vtt）または「テキストファイル」（.txt）をダウンロード
4. この画面にファイルをドラッグ＆ドロップ、または「ファイルを選択」ボタンをクリック

【対応形式】
✓ .vtt（WebVTT形式）
✓ .txt（テキスト形式）

【ファイルサイズ上限】
50MBまで
```

### アップロード中

```
【プログレス表示】
📤 アップロード中... [==========>    ] 65%

【メッセージ】
ファイルをアップロードしています。しばらくお待ちください。
```

### 処理中

```
【プログレス表示】
🤖 AIが議事録を作成中...（約30秒〜1分）

【メッセージ】
文字起こしデータを分析し、議事録を生成しています。
この処理は自動で行われます。ページを閉じないでください。
```

### エラー時

```
【エラータイプ別メッセージ】

1. ファイル形式エラー
   "アップロードされたファイルの形式が非対応です。VTT(.vtt)またはTXT(.txt)ファイルを選択してください。"

2. ファイルサイズ超過
   "ファイルサイズが大きすぎます（上限50MB）。ファイルを分割するか、圧縮してから再度アップロードしてください。"

3. 処理エラー
   "議事録の作成中にエラーが発生しました。ファイルの内容を確認の上、再度アップロードしてください。問題が続く場合はお問い合わせください。"

4. 空ファイル
   "アップロードされたファイルが空です。内容を確認の上、再度アップロードしてください。"
```
