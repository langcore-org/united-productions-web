## 目的

過去に開発した対話型新規企画立案機能（neta-researcher）を、現在のアプリ（Teddy/AI Hub）に移植する。

## 背景

- United Productions Web に「ネタ企画作家」モードとして実装済み

## 実装方針

### 基本方針

```
過去アプリのプロンプト → Grok向けに調整 → DB登録 → UI実装 → 動作確認
```

### プロンプト設計

**使用プロンプトキー**: `PROPOSAL`（既存）または `NEW_PLANNING`（新設）

現在の `PROPOSAL` キーのプロンプトをneta-researcherベースに書き換えるか、新しいキー `NEW_PLANNING` を作成する選択肢があります。

**推奨**: `PROPOSAL` を拡張（既存機能との整合性を保つため）

### 実装アプローチ

```
Phase 1: 最小限実装（MVP）
├── プロンプト調整・DB登録
├── Google Drive連携API実装（ファイル保存用）
└── シンプルなリサーチ → レポート生成フロー

Phase 2: 機能強化（3月12日以降）
├── タスクリストUI（フロントエンド管理）
├── 画像収集・表示機能
└── レポートテンプレート改善

Phase 3: 高度な機能（将来）
├── WebFetch相当の詳細情報取得
├── インフォグラフィック生成
└── 複数フォーマット出力（PDF等）
```

## 仕様

### 機能名
- **正式名称**: 「新企画立案」（暫定）
- **Sidebar表示名**: 「新規企画立案」または「企画立案」
- **サブテキスト**: 「アイデアを出してくれます」（案）

### 動作イメージ（MVP版）

```
1. ユーザー: Sidebarから「新規企画立案」を選択
2. AI: 「お疲れ様です！どのような番組企画をお考えですか？ジャンルやテーマがあれば教えてください。」
3. ユーザー: 「バラエティで、若者向けの新しい企画を考えたい」
4. AI: （web_searchツールで自動検索）「現在の若者向けバラエティトレンドを調べます...」
5. AI: 「調査結果に基づき、以下の企画案をご提案します...」（レポート形式で出力）
6. AI: 「レポートをGoogle Driveに保存しました。リンク：[Driveリンク]」
```

### 出力内容（MVP）

- 企画タイトル案
- コンセプト説明
- 進行イメージ
- 想定出演者（候補）
- ターゲット層
- 放映枠の提案
- 参考になった情報源

### 出力形式（レポート）

neta-researcherの形式を流用：

```markdown
# リサーチレポート: [タイトル]

## エグゼクティブサマリー
- [ネタの概要を一言で]

## 主要なインサイト (3-5個)
### Insight 1: [キャッチーな見出し]
- **発見の背景**: [なぜこれが面白いか]
- **裏付けデータ**: [数字・事実]
- **参考リソース**: [URL]
- **構成案**: [VTRのイメージ]

## 参考資料リスト
[テーマごとに分類]
```
