# テストから本番への移行

> **優先度**: 🟡 中  
> **期限**: 2026-03-10  
> **担当**: 浅野宏耀  
> **ステータス**: 未着手

## 目的

Google認証をテストモードから本番モードに移行し、テストユーザー登録なしで誰でもログインできるようにする。

## 現状

- 現在はテストモード（Google Cloud Console）
- テストユーザーとして登録しないとログインできない状態
- 3月12日の現場テスト前には本番移行が必要

## 移行手順

### 1. Google Cloud Console設定変更（1日）

- [ ] OAuth同意画面の設定確認
- [ ] 本番モードへの切り替え
- [ ] 必要なスコープの確認
- [ ] 承認リクエストの送信（必要な場合）

### 2. 環境変数更新（0.5日）

- [ ] `.env.local`の確認
- [ ] 本番環境変数の設定
- [ ] Vercel環境変数の更新

### 3. 動作確認（0.5日）

- [ ] テストユーザーでログイン確認
- [ ] 新規ユーザー登録フローの確認
- [ ] セッション管理の確認

## リスクと対策

| リスク | 対策 |
|--------|------|
| Googleの承認に時間がかかる | 3月初頭に早めに申請 |
| スコープ変更が必要 | 必要最小限のスコープに抑える |
| ドメイン認証 | カスタムドメインの確認 |
