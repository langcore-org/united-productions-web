# Vercel AI SDK 導入計画書

> **Next.jsプロジェクトへのVercel AI SDK段階的導入計画**
>
> **作成日**: 2026-02-21 19:00
> **調査基準日**: 2026-02-21（AI SDK v4安定版、v6ベータ版情報を含む）

---

## 1. 計画概要

### 1.1 目的

現在の独自実装（`lib/llm/`）における以下の課題を解決する：

- ストリーミング処理の複雑さ（SSE生成、エラーハンドリング）
- チャットUIの状態管理（スクロール制御、メッセージ更新）
- ツール呼び出しの標準化
- マルチプロバイダー対応の工数

### 1.2 方針

**Vercel AI SDKを段階的に導入し、複雑なエージェント機能が必要になった時点でLangChain併用を検討する。**

## 2. Vercel AI SDK 概要（最新情報）

### 2.1 バージョン情報

| バージョン | ステータス | 主な特徴 |
|-----------|-----------|---------|
| v4.x | 安定版（推奨） | ストリーミング、ツール呼び出し、React Hooks |
| v6.x | ベータ版 | Agent抽象化、人間承認フロー（`needsApproval`） |

### 2.2 構成要素

#### AI SDK Core（バックエンド）
- **用途**: テキスト生成、ストリーミング、ツール呼び出し
- **主要関数**:
  - `generateText`: 同期的テキスト生成
  - `streamText`: ストリーミングテキスト生成（SSE対応）
  - `generateObject`: 構造化データ生成
  - `streamObject`: ストリーミング構造化データ

#### AI SDK UI（フロントエンド）
- **用途**: フレームワーク非依存のReact Hooks
- **主要フック**:
  - `useChat`: チャット状態管理（メッセージ、入力、ローディング、エラー）
  - `useCompletion`: テキスト補完管理
  - `useObject`: ストリーミングJSONオブジェクト消費

### 2.3 サポートフレームワーク

| フレームワーク | パッケージ名 | 対応フック |
|--------------|-------------|-----------|
| React | `@ai-sdk/react` | useChat, useCompletion, useObject |
| Vue.js | `@ai-sdk/vue` | useChat, useCompletion, useObject |
| Svelte | `@ai-sdk/svelte` | Chat, Completion, StructuredObject |
| Angular | `@ai-sdk/angular` | Chat, Completion, StructuredObject |

## 3. 導入メリット

### 3.1 技術的メリット

| 項目 | 現状（独自実装） | Vercel AI SDK導入後 |
|------|---------------|-------------------|
| ストリーミング実装 | SSE生成、エラーハンドリング自前実装 | `streamText()` 1行で完了 |
| チャット状態管理 | useState/useEffect複雑制御 | `useChat`フックで自動管理 |
| ツール呼び出し | パース、実行、結果返却を自前実装 | 宣言的定義で自動実行 |
| スクロール制御 | 自前で実装（バグリスク） | 自動スクロール制御付き |
| エラーハンドリング | 自前実装 | 組み込みエラーリトライ機能 |

### 3.2 生産性メリット

- **学習コスト**: 緩やか（Next.js開発者に親和性高）
- **実装工数**: ストリーミング機能で約70%削減見込み
- **バグ削減**: 標準化されたパターンで予期せぬ挙動を防止
- **メンテナンス**: Vercel社公式サポートで長期安定性確保

## 4. 移行フェーズ

### Phase 0: 準備（1-2週間）

#### タスク
- [ ] 現状の`lib/llm/`実装の機能一覧作成
- [ ] テスト環境構築（Vercel AI SDK動作確認用）
- [ ] チーム内知識共有（ドキュメント読み合わせ）

#### 成果物
- 現状機能マッピングドキュメント
- 技術検証サンプルコード

---

### Phase 1: パイロット導入（2-4週間）

#### タスク
- [ ] 新規機能（または影響の少ない機能）からVercel AI SDKを採用
- [ ] API Routesの一部を`streamText`に置き換え
- [ ] フロントエンドの一部を`useChat`に置き換え
- [ ] 動作検証とパフォーマンス測定

#### 対象機能候補
- 新規チャット機能
- プロトタイプ機能
- 管理画面のAI補助機能

#### 成果物
- パイロット機能の実装
- パフォーマンス比較レポート
- 移行ガイドライン（社内用）

---

### Phase 2: 段階的移行（1-2ヶ月）

#### タスク
- [ ] 既存機能を優先度順に移行
- [ ] 高優先度：ストリーミング関連のバグ修正が必要な機能
- [ ] 中優先度：新規要求のある機能
- [ ] 低優先度：現状で安定動作している機能

#### 移行優先度基準
| 優先度 | 基準 |
|-------|------|
| 高 | ストリーミングバグあり、メンテナンス工数大 |
| 中 | 機能追加要求あり、将来の拡張性必要 |
| 低 | 現状安定、変更リスク大 |

#### 成果物
- 移行済み機能一覧
- レグレッションテスト結果

---

### Phase 3: 全面移行（2-3ヶ月）

#### タスク
- [ ] 残存の独自実装をVercel AI SDKに完全移行
- [ ] `lib/llm/`はプロバイダー設定・カスタム設定のみに簡略化
- [ ] レガシーコードの削除
- [ ] ドキュメント更新

#### 成果物
- 完全移行したコードベース
- 更新された技術仕様書
- 運用マニュアル

---

### Phase 4: 高度化（必要に応じて）

#### 検討タイミング
- 複雑なエージェントワークフロー（多段階推論、自律的タスク実行）が必要になった時
- AI SDK v6（Agent抽象化）が安定版になった時

#### 選択肢
1. **Vercel AI SDK v6へアップグレード**
   - Agent抽象化（`ToolLoopAgent`）
   - 人間承認フロー（`needsApproval`）

2. **LangChain併用**
   - 複雑なエージェント機能のみLangChainで実装
   - シンプルなチャットはVercel AI SDK維持

## 5. リスクと対策

### 5.1 技術的リスク

| リスク | 内容 | 対策 |
|--------|------|------|
| 破壊的変更 | AI SDKのメジャーアップデート | 安定版（v4）を使用、段階的アップデート |
| 機能不足 | 独自実装の特殊機能が未サポート | ラッパー関数で拡張、または移行対象外とする |
| パフォーマンス劣化 | 抽象化レイヤーによるオーバーヘッド | パイロットで計測、問題あれば最適化 |

### 5.2 運用リスク

| リスク | 内容 | 対策 |
|--------|------|------|
| 学習コスト | チーム全体の習得時間 | ドキュメント整備、ペアプログラミング |
| 移行中断 | 他優先タスクによる中断 | 段階的アプローチで影響最小化 |
| 品質低下 | 移行によるバグ混入 | 十分なテスト、段階的リリース |
