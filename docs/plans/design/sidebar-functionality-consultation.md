## 背景

会議冒頭で「前回話したサイドバー機能について、具体的な機能の実装方法について相談したい」という意向が示された。

## 第1段階リリースのサイドバー構成

```
【リリース対象機能】
1. チャット
2. 出演者リサーチ
3. エビデンスリサーチ
4. 議事録作成
5. 新企画立案
6. 履歴
```

※ナレーション原稿作成は優先度を下げたため、第1段階では非表示

## 相談事項（想定）

### 1. サイドバーの構成について

**検討点**:
- 機能の配置順序
- グルーピング（カテゴリ分け）の要否
- アイコンのデザイン統一

**提案案**:

```
案A: 機能別グルーピング
├─ チャット
├─ リサーチ
│   ├─ 出演者リサーチ
│   └─ エビデンスリサーチ
├─ 制作支援
│   ├─ 議事録作成
│   └─ 新企画立案
└─ 履歴

※ナレーション原稿作成は第2段階以降に実装予定

案B: シンプルリスト（推奨）
├─ チャット
├─ 出演者リサーチ
├─ エビデンスリサーチ
├─ 議事録作成
├─ 新企画立案
└─ 履歴

※ナレーション原稿作成は第2段階以降に実装予定
```

### 2. 機能へのアクセス方法について

**検討点**:
- サイドバーから直接機能を開くか
- チャットからエージェントを選択する形式か

**現在の実装**:
- チャットをトップに配置
- リサーチ等は「エージェント」として選択可能

### 3. UI/UXの詳細

**検討点**:
- アクティブ状態の表示方法
- ホバー効果
- 新規機能のバッジ表示（リリース後）
- 折りたたみ機能の要否
