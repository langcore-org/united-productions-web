## テスト項目

### 1. 出演者リサーチ

**テストシナリオ**:

```
シナリオ1: 有名タレントのリサーチ
- 入力: 「[有名タレント名]をリサーチして」
- 確認項目:
  □ 基本情報が正確に表示されるか
  □ SNSアカウント情報が正確か
  □ 出演歴にUnited Productions関連番組が含まれるか
  □ 出典リンクが正しく機能するか
  □ 応答時間は許容範囲か（30秒以内）

シナリオ2: 中堅タレントのリサーチ
- 入力: 「[中堅タレント名]をリサーチして」
- 確認項目:
  □ 情報の網羅性
  □ 最新情報が含まれているか
  □ 出力形式の一貫性
```

**評価基準**:
- ⭐⭐⭐ 現場で即座に使える
- ⭐⭐☆ 少し修正が必要
- ⭐☆☆ 大幅な改善が必要

### 2. エビデンスリサーチ

**テストシナリオ**:

```
シナリオ: 企画テーマに関するエビデンス収集
- 入力: 「[テーマ]に関するエビデンスを収集して」
- 確認項目:
  □ 出典が明確に示されているか
  □ 信頼性の高い情報源が使われているか
  □ 統計データが含まれているか
  □ 具体的事例が含まれているか
```

### 3. 議事録作成

**テストシナリオ**:

```
シナリオ1: VTTファイルのアップロード
- 操作: ZoomのVTTファイルをアップロード
- 確認項目:
  □ アップロードがスムーズに行えるか
  □ 処理時間は許容範囲か
  □ 生成された議事録の品質
  □ 決定事項が正しく抽出されているか
  □ アクショナイテムが抽出されているか

シナリオ2: TXTファイルのアップロード
- 操作: テキストファイルをアップロード
- 確認項目:
  □ 同様の品質で処理されるか
```

### 4. 新企画立案

**テストシナリオ**:

```
シナリオ: 新番組企画のブレスト
- 操作: 「新しいバラエティ番組を考えたい」と入力
- 確認項目:
  □ 対話が自然に進むか
  □ アイデアの質はどうか
  □ 企画案の構成は適切か
  □ 想定出演者の提案は適切か
```

### 5. 全般的なUI/UX

**確認項目**:

- [ ] 画面の見やすさ
- [ ] 操作の直感性（説明なしで使えるか）
- [ ] 応答速度
- [ ] エラー発生時の挙動
- [ ] スマートフォンでの表示（必要に応じて）
