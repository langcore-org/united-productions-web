## PJ-A: 議事録・文字起こしツール

### 目的
Zoom文字起こしテキストをアップロードすると、議事録フォーマットに自動整形するウェブツール。

### 2〜4月ロードマップとの関係
2〜4月はまずZoom文字起こし＋Gemini/Claude手動整形の運用で対応する（ツール導入）。4月の効果測定で**手動整形の工数が依然として大きい**と判断された場合にLC開発に着手する。

### LC開発時の機能要件
- **入力:** Zoom文字起こしテキスト or 録音ファイル（mp3, m4a, wav, mp4）をアップロード
- **高精度文字起こし（録音入力時）:** Whisper APIで音声→テキスト変換
- **話者分離（録音入力時）:** pyannote等で話者を自動分離
- **AI整形:** Claude/Gemini APIで議事録フォーマットに整形（議題・発言要旨・決定事項・TODO）
- **テンプレート:** 会議用・面談用（人物名・経歴・話した内容・出演可否）の2パターン
- **出力:** Word / Googleドキュメント / Markdown

### 技術スタック
- **フロントエンド:** Next.js / React（アップロードUI + プレビュー + 編集）
- **バックエンド:** Python FastAPI
- **音声処理:** OpenAI Whisper API, pyannote.audio, ffmpeg
- **AI:** Claude API / Gemini API
- **ドキュメント生成:** python-docx
- **認証:** Google Workspace SSO

### 備考
PJ-Bと音声処理基盤（Whisper + pyannote）を共有可能。両方着手する場合は共通基盤として構築する。

## PJ-B: 起こし・NA原稿ツール

### 目的
映像の音声から文字起こしを行い、話者分け＋タイムコード付きの起こし・NA原稿を既存Wordフォーマットで出力するツール。

### 2〜4月ロードマップとの関係
2〜4月はまずPremiere Pro内蔵AI / Nottaで検証する（ツール導入）。3月の検証結果で**既存ツールでは精度・フォーマットが不十分**と判断された場合にLCモック検討に着手する。

### LC開発時の機能要件
- **入力:** 映像ファイル（mp4等）→ 音声トラック自動抽出
- **文字起こし:** Whisper APIで高精度変換
- **話者分離・ラベル推定:** pyannoteで話者分離し、AIが「櫻井」「末澤」「N」等のラベルを推定
- **ナレーション区別:** 「N」（ナレーション）を自動判定
- **タイムコード付与:** 自動的にタイムコードを付与
- **Word出力:** 既存の台本フォーマット（箱型レイアウト）に準拠したWordファイルをpython-docxで生成

### 技術スタック
- PJ-Aと同一（音声処理基盤を共有）
- **追加:** python-docx（台本フォーマットWord出力）、ffmpeg（動画→音声抽出）

## PJ-C: リサーチ・考査チャットアプリ

### 目的
専門家候補探し（No.1）・ロケ地探し（No.2）・考査エビデンス確認（No.20）を、ガイド付きチャットUIで一括対応するウェブアプリ。社内ドライブの過去資料も横断参照できるようにする。

### 2〜4月ロードマップとの関係
**2〜4月のメインLC開発対象。** 2月後半に要件定義、3月に設計・開発着手、4月にα版テスト。

### アーキテクチャ概要
```
[ユーザー] → [ガイド付きチャットUI（選択肢誘導）]
                    ↓
            [ルーティングレイヤー]
           ↙        ↓         ↘
     [Grok API]  [Perplexity API]  [Google Drive API]
     (SNS/X検索)  (Web検索+ソース)   (社内資料検索)
                    ↓
         [結果整形 → リスト出力 / ソース付き回答]
```

### 機能要件
- **ガイド付きUI:** 「何をリサーチしますか？」→「人探し / 場所探し / エビデンス確認」→ サブカテゴリ → 条件入力
- **人探しメニュー（α版スコープ）:** SNS・Web上の最新情報を検索し、候補リスト（30人程度）を自動生成。名前・プロフィール概要・URL・SNSリンクを表形式で表示。CSV/Excelダウンロード可能
- **場所探しメニュー:** 社内ドライブの過去ロケ地資料を横断検索。条件（エリア・用途・予算帯）で絞り込み
- **エビデンス確認メニュー:** Perplexityでソース付き回答。考査提出用にエビデンスURLを一覧出力
- **プロンプトプリセット:** よく使うリサーチパターン（「○○マニア探し」「○○のエビデンスを探して」等）をプリセットとして用意
- **社内ドライブ参照:** Google Drive APIで集約済みの過去リサーチ資料・ロケ地情報を検索対象に含める

### 技術スタック
- **フロントエンド:** Next.js / React（チャットUI + 選択肢コンポーネント）
- **バックエンド:** Python FastAPI（APIルーティング・結果整形）
- **外部API:** Grok API（SNS/Xリアルタイム検索）、Perplexity API（Web検索+ソース付き）、Google Drive API（社内資料検索）
- **認証:** Google Workspace SSO

### 将来の拡張（5月以降）
- キャスティングメニュー追加（No.3）
- 飲食店・スポットメニュー追加（No.4 / Google Places API連携）
- 社内ナレッジ検索の本格版（Vertex AI Search）へのアップグレード

## PJ-E: 台本・制作資料ジェネレーター（5月以降）

### 目的
構成資料のプロットや既存資料を入力として、各種制作資料の叩き台をWordフォーマットで自動生成する。

### 統合する業務ソリューション
| ID | 業務名 | 統合の形 |
|----|--------|---------|
| 8-a | ロケ台本自動生成ツール | コア機能として初期リリース |
| 18-a | スタジオ収録台本 | 拡張モード |
| 16-a | カンペ自動抽出 | 台本からの演者別セリフ抽出機能 |
| 15-a | テロップ原稿自動生成 | 起こし原稿（PJ-B）からの連携 |
| 22-b | 視聴者対応マニュアル自動生成 | 台本・起こしからの自動要約 |

### 機能要件
- **ロケ台本モード:** テーマ・ロケ先・出演者を入力 → 既存Wordフォーマットにセリフ案・ナレーション案を流し込み
- **Wordテンプレート登録:** 番組ごとのフォーマットをテンプレートとして管理
- **編集・再生成:** ブラウザ上でプレビュー・編集、部分的に再生成可能
- **追加モード:** スタジオ台本 / カンペ抽出 / テロップ原稿 / 視聴者対応マニュアル

### 技術スタック
- **フロントエンド:** Next.js / React
- **バックエンド:** Python FastAPI
- **AI:** Claude API / Gemini API
- **ドキュメント生成:** python-docx, openpyxl
- **認証:** Google Workspace SSO

## PJ-F: ロケスケ連動テンプレート（GAS版）（5月以降）

### 目的
ロケスケジュール・香盤表・車両表・演者別版を連動させ、マスターの1箇所を修正すれば全シートに反映される仕組みを**Google スプレッドシート + GAS**で構築する。2〜4月のPJ-D（ウェブアプリ）の代替案として、GASのみで完結する選択肢。

### 機能要件
- **マスターシート:** ロケスケジュール（全体版）に時間軸・演者・スタッフ・場所を入力
- **自動生成:** 演者別版・香盤表・車両表をマスターから自動生成
- **連動更新:** マスター変更が全シートに自動反映
- **車両台数自動計算:** 同時刻に必要な車両数を自動算出

### 技術スタック
- **プラットフォーム:** Google スプレッドシート + Google Apps Script（GAS）

### 備考
他PJへの技術的依存がなく、GASのみで完結する。ロケスケは2〜4月でPJ-Dのウェブアプリを先行開発するため、本PJ-Fは5月以降の追加選択肢。

## PJ-G: 校閲支援ツール（5月以降）

### 目的
テロップ原稿の誤字脱字・漢字ミス・表記ゆれをAIで自動検出するウェブツール。

### 機能要件

- 誤字脱字・漢字ミス・表記ゆれを自動検出し、修正箇所をハイライト表示

- チェック履歴保存・エクスポート

### 技術スタック
- **フロントエンド:** Next.js / React
- **バックエンド:** Python FastAPI
- **AI:** Claude API
- **データベース:** PostgreSQL
- **認証:** Google Workspace SSO

### 備考
考査チェック（No.20）は PJ-C のチャットアプリに統合済み。本ツールは校閲（No.19）に特化する。

|--------|------|------|
| ドライブへの資料集約が進まない | PJ-Cのドライブ参照機能が使えない | 2月後半で集約ルール策定、3月にUPから号令。1番組分で先行 |

| スタッフのツール移行抵抗 | 導入しても使われない | 2〜4月のツール導入で成功体験を作り、ハンズオン研修で定着 |
