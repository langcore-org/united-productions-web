## 1. 概要・目的

専門家探し（No.1）・ロケ地探し（No.2）・考査エビデンス確認（No.20）を**1つのチャットアプリに統合**する。Grok・Perplexity・社内ドライブを参照し、ガイド付きUIで「人探し / 場所探し / エビデンス確認」を一括対応する。

**目的:**
- 専門家（マニア）候補のリサーチ: 外注・手動検索の工数削減（候補リスト一括生成→人間で選別）
- ロケ場所のリサーチ: 社内ドライブに集約した過去ロケ地資料の横断検索
- 考査エビデンス: 局考査部提出用の出典・根拠をソース付きで確認

**2〜4月ゴール（ロードマップ準拠）:** α版で**人探しメニュー**が動作し、実案件で試用開始。4月末時点で「人探し」を先行リリースし、場所探し・エビデンス確認は設計済みまたは部分実装とする。

## 3. アーキテクチャ概要

```
[ユーザー] → [ガイド付きチャットUI（選択肢＋自由入力）]
                    ↓
            [ルーティング / 意図判定]
           ↙        ↓         ↘
     [Grok API]  [Perplexity API]  [Google Drive API]
     (SNS/X検索)  (Web検索+ソース)   (社内資料検索)
                    ↓
         [結果整形 → リスト出力 / ソース付き回答 / チャット表示]
```

- **フロント:** チャットUI ＋ メニュー選択（人探し / 場所探し / エビデンス確認）＋ 条件入力フォーム
- **バックエンド:** メニュー・クエリに応じて Grok / Perplexity / Drive のいずれかまたは組み合わせを呼び出し、結果を整形して返す
- **認証:** Google Workspace SSO（社内ドライブ参照のため）

## 4. 機能要件

### 4.1 共通

| # | 機能 | 内容 |
|---|------|------|
| F0-1 | ガイド付きUI | 初回または「やり直し」で「何をリサーチしますか？」と表示し、「人探し」「場所探し」「エビデンス確認」を選択させる |
| F0-2 | プロンプトプリセット | よく使うパターン（「○○マニア探し」「○○のエビデンスを探して」等）をボタンまたはプルダウンで選択可能 |
| F0-3 | 認証 | Google Workspace SSO。Drive 参照時は同一ワークスペースのドライブのみアクセス可能とする |

### 4.2 人探しメニュー（α版で完成）

| # | 機能 | 内容 |
|---|------|------|
| F1-1 | 条件入力 | テーマ・キーワード（例: 「〇〇マニア」「〇〇に詳しい人」）を入力。必要に応じて番組ジャンル・人数目安を選択 |
| F1-2 | 検索実行 | Grok API で SNS/X の最新情報を検索。必要に応じて Perplexity API で Web 上の記事・プロフィールを補完 |
| F1-3 | 結果表示 | 候補リスト（30人程度を目安）を表形式で表示。項目: 名前・プロフィール概要・URL・SNSリンク等 |
| F1-4 | エクスポート | リストを CSV / Excel でダウンロード可能 |

### 4.3 場所探しメニュー（α版では設計＋部分実装）

| # | 機能 | 内容 |
|---|------|------|
| F2-1 | 条件入力 | エリア・用途・予算帯・キーワード（「過去に使った場所」「面倒な店主」等）を入力 |
| F2-2 | 社内ドライブ検索 | Google Drive API で集約済みの過去ロケ地資料を検索。フォルダ構成はドライブ集約ルールに準拠 |
| F2-3 | 結果表示 | ヒットした資料・スニペットをチャット内に表示。必要なら Perplexity で Web 上の場所情報を補完 |

### 4.4 エビデンス確認メニュー（α版では設計＋部分実装）

| # | 機能 | 内容 |
|---|------|------|
| F3-1 | クエリ入力 | 「この情報の根拠となる出典を出して」等、考査用の質問を入力 |
| F3-2 | ソース付き検索 | Perplexity API でソース付き回答。Grok で SNS 上の発言の裏取りが必要な場合は併用 |
| F3-3 | 結果表示 | 回答とともにエビデンス URL を一覧表示。考査提出用にコピーしやすい形式 |

## 5. データ・API設計

### 5.1 外部API

| API | 用途 | 主な入力 | 主な出力 |
|-----|------|----------|----------|
| Grok API | SNS/X リアルタイム検索、専門家・トレンド候補の収集 | 検索クエリ（テーマ・キーワード） | 投稿・プロフィール・リンク等のテキスト/構造 |
| Perplexity API | Web 検索＋ソース付き回答、考査エビデンス | 質問文・検索クエリ | 回答テキスト＋引用URL一覧 |
| Google Drive API | 社内のリサーチ資料・ロケ地資料の検索 | 検索クエリ、フォルダID（スコープ） | ファイルメタデータ・テキスト抽出（Drive の検索機能またはドキュメント内容取得） |

### 5.2 バックエンドAPI（アプリ側）

- **POST /chat** または **/api/message**: ユーザー発話＋選択メニュー（人探し/場所探し/エビデンス）を受け取り、ルーティング後に Grok/Perplexity/Drive を呼び出し、整形結果を返す。
- **GET /api/export?session=xxx&format=csv|excel**: 直近の「人探し」結果をCSV/Excelでダウンロード（セッションまたは会話IDで紐付け）。
- 認証: Google OAuth 2.0 によるトークン取得。Drive 利用時はスコープに `drive.readonly` 等を含める。

### 5.3 ドライブ集約ルールとの関係

- チャットアプリが参照するドライブのフォルダ構成・命名ルールは「ドライブ集約ルール」に従う。LC からフォルダ構成案を提示し、UP から号令を出す想定（ロードマップ 2月後半〜3月）。
- 実装では「検索対象フォルダID」を設定で持つか、共通の「リサーチ用ルートフォルダ」を前提とする。

## 6. ユーザーフロー（α版：人探し中心）

1. ユーザーがアプリにアクセスし、Google SSO でログイン。
2. 初回メッセージまたは「新規リサーチ」で「何をリサーチしますか？」が表示され、「人探し」「場所探し」「エビデンス確認」を選択。
3. **人探し**を選択した場合:
   - 「テーマ・キーワードを入力してください」等のプロンプトが表示される。
   - ユーザーが「〇〇マニア」「〇〇に詳しい人」等を入力。
   - バックエンドが Grok（＋必要なら Perplexity）を呼び出し、候補リストを整形。
   - チャットに表形式で結果表示。ボタン「CSVダウンロード」「Excelダウンロード」でエクスポート。
4. **場所探し**を選択した場合（α版では簡易対応または「準備中」表示）:
   - 条件入力後、Drive 検索（＋必要なら Perplexity）を実行し、ヒットした資料・スニペットを表示。
5. **エビデンス確認**を選択した場合（α版では簡易対応または「準備中」表示）:
   - 質問入力後、Perplexity でソース付き回答を取得し、URL一覧とともに表示。

## 7. 技術スタック

| レイヤー | 技術案 | 備考 |
|----------|--------|------|
| **フロントエンド** | Next.js / React | チャットUI＋選択肢コンポーネント。PJ-D（ロケスケ）等他LCと揃える場合は React 系で統一 |
| **バックエンド** | Python FastAPI | APIルーティング・Grok/Perplexity/Drive の呼び出し・結果整形。他LCが FastAPI の場合は合わせる |
| **外部API** | Grok API, Perplexity API, Google Drive API | 各プロバイダの SDK または REST クライアント |
| **認証** | Google Workspace SSO（OAuth 2.0） | ドライブ参照のため。フロントまたはバックエンドでトークン取得・検証 |
| **デプロイ** | 要検討（オンプレ / クラウド） | 社内利用前提であれば社内サーバーまたは Google Cloud 等 |

※ ロケスケ実装プランでは「他LC（PJ-C等）が FastAPI を採用している場合は合わせる」とあるため、PJ-C を FastAPI で統一すると他PJとの共通基盤を取りやすい。

## 9. リスク・前提条件

| リスク | 対応 |
|--------|------|
| ドライブへの資料集約が進まない | 2月後半で集約ルール策定、3月に UP から号令。1番組分から先行してドライブに格納する運用を開始 |
| Grok API の利用可否・レート制限 | 早期に API 契約・利用条件を確認。暫定として Grok（X Premium）の直接利用で候補リスト生成の運用を並行可能（業務洗い出し 1-b） |
| Perplexity API のコスト・制限 | 利用量をモニタリングし、α版テスト期間でコスト見通しを取得 |
| 考査エビデンスの精度 | ソース付き回答を「補助」とし、最終判断は人間が行う前提で周知 |
