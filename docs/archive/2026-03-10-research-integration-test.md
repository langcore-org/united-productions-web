## 目的

出演者リサーチとエビデンスリサーチの機能を統合し、3月12日の現場テストに向けて安定した状態にする。

## テスト範囲

### 統合テスト対象機能

| 機能 | テスト項目 |
|------|-----------|
| **出演者リサーチ** | 検索実行 → 結果表示 → 出典リンク確認 |
| **エビデンスリサーチ** | 検索実行 → 出典表示 → 結果コピー |
| **チャット連携** | リサーチ結果をチャットに引き継ぎ |
| **履歴機能** | リサーチ履歴の保存・再表示 |

## テストシナリオ

### シナリオ1: 基本フロー

```
前提: ユーザーがログイン済み

1. サイドバーから「出演者リサーチ」を選択
2. 検索欄に「有名人A」と入力
3. 「検索」ボタンをクリック
4. 結果が表示される（30秒以内）
5. 結果に以下が含まれる:
   - 基本プロフィール
   - SNSアカウント（X、Instagram等）
   - 出演歴
   - 出典リンク
6. 出典リンクをクリック
7. 正しいページが開く
8. チャットに戻る
9. 履歴からリサーチ結果を再表示
```

### シナリオ2: エラーハンドリング

```
1. 検索欄に不適切な入力（記号のみ等）
2. エラーメッセージが表示される
3. 再検索が可能
```

### シナリオ3: エビデンスリサーチ連携

```
1. 出演者リサーチで「有名人A」を検索
2. エビデンスリサーチで「有名人Aのトレンド」を検索
3. 両方の結果が整合していることを確認
```

## テスト項目詳細

### 機能テスト

| ID | テスト項目 | 期待結果 | 優先度 |
|----|-----------|----------|--------|
| R-001 | 検索実行 | 30秒以内に結果表示 | P0 |
| R-002 | 結果表示 | 指定した形式で表示 | P0 |
| R-003 | 出典リンク | 正しいページに遷移 | P0 |
| R-004 | 履歴保存 | 検索履歴が保存される | P1 |
| R-005 | 履歴再表示 | 履歴から結果を再表示 | P1 |
| R-006 | エラー表示 | 適切なエラーメッセージ | P1 |
| R-007 | 長時間検索 | 60秒以上かかる場合の処理 | P2 |

### パフォーマンステスト

| ID | テスト項目 | 目標値 |
|----|-----------|--------|
| P-001 | 初回検索応答時間 | ≤30秒 |
| P-002 | 2回目以降の検索 | ≤20秒（キャッシュ活用） |
| P-003 | 同時検索（負荷テスト） | 3件同時実行でエラーなし |

### UI/UXテスト

| ID | テスト項目 | 確認内容 |
|----|-----------|----------|
| U-001 | 思考プロセス非表示 | AIの思考過程が表示されない |
| U-002 | 自動スクロール制御 | ユーザー操作時に自動スクロールしない |
| U-003 | レスポンシブ表示 | 画面サイズに応じた表示 |
