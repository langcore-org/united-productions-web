## 背景

社内朝礼でのスクリーンショット提供時に、新企画名「新企画立案」のサブテキスト（説明文）の内容についても考慮する必要がある。

## サブテキストの役割

- 機能名だけでは伝わりにくい意図を補足
- ユーザーがその機能で何ができるかを簡潔に説明
- UI上での視覚的な階層（機能名＞サブテキスト）

## 候補案

### 案1: アイデア創出系

| サブテキスト | 評価 |
|--------------|------|
| アイデアを出してくれます | ⭐ シンプルで分かりやすい |
| 新しい企画のアイデアを一緒に考えます | やや長いが具体的 |
| ブレインストーミングのお手伝い | 「ブレスト」よりは一般的 |

### 案2: 対話・ディスカッション系

| サブテキスト | 評価 |
|--------------|------|
| 企画のディスカッションができます | 具体的 |
| AIと一緒に企画を練ります | AI活用を明示 |
| 対話しながら企画を磨きます | プロセスを重視 |

### 案3: 提案・創作系

| サブテキスト | 評価 |
|--------------|------|
| 企画案を提案してくれます | 結果重視 |
| 企画のヒントをくれます | 親しみやすい |
| 新企画を一緒に立案します | 機能名と重複しがち |

## 確定案

**「アイデアを出してくれます」**

理由：
- シンプルで分かりやすい
- 機能の本質（アイデア創出）を端的に表現
- 他の機能（「エビデンスリサーチ」等）との統一感がある
- サブテキストとして適切な長さ
