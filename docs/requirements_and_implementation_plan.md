## 1. プロジェクト概要

### 1.1 目的
AD（アシスタントディレクター）業務を支援するAIツールを提供し、制作業務の効率化を実現する。

### 1.2 ターゲットユーザー
- AD（アシスタントディレクター）
- ディレクター
- プロデューサー

### 1.3 リリーススケジュール
- **3月末リリース**（2月末はスケジュール的に厳しいため延期）

## 2. 機能要件

### 2.1 3月末リリース機能（優先実装）

| 機能名 | 説明 | カテゴリ |
|--------|------|----------|
| **NA原稿作成** | Premiere Pro書き起こしテキストからナレーション原稿を生成 | テキスト整形 |
| **議事録作成** | Zoom/打ち合わせの文字起こしから構造化された議事録を生成 | テキスト整形 |
| **出演者リサーチ** | 企画に適した出演者候補をネット上からリサーチ | リサーチ |

| **情報リサーチ** | テーマに関する情報・雑学・トリビアを収集 | リサーチ |
| **エビデンスリサーチ** | 情報の真偽検証・ファクトチェック | リサーチ |
| **新企画立案** | 番組特性を考慮した新しい企画案を提案 | 企画立案 |

## 3. 詳細仕様

### 3.1 NA原稿作成

#### 入力
- Premiere Pro書き起こしテキスト（.txt, .vtt形式）
- **音声入力は一旦見送り**（データ容量・運用費用の問題）

#### 出力
- ナレーション原稿（Word形式でダウンロード可能）
- 話者ラベル付き（櫻井/末澤/N/その他）
- タイムコード除去済み
- フィラー（えー、あーなど）除去済み

#### プロセス
1. 書き起こしテキストをアップロード/貼り付け
2. AIが整形・構造化
3. プレビュー表示
4. Word形式でダウンロード

### 3.2 議事録作成

#### 入力
- Zoom文字起こしテキスト
- 打ち合わせのテキスト

#### 出力
- 構造化された議事録
  - 会議概要（テーマ、参加者、日時）
  - 議題別まとめ
  - 決定事項
  - TODO・担当者
  - 次回までの課題

#### テンプレート
- **会議用**: 一般的な会議議事録形式
- **面談用**: 出演者面談用（プロフィール、出演意向など）

### 3.3 リサーチ機能（4つ）

| リサーチタイプ | 参照データ | 裏で動くAI |
|----------------|-----------|-----------|
| **出演者リサーチ** | オンライン情報 + 社内過去データ | Grok（X検索強み） |
| **場所リサーチ** | オンライン情報 + 社内過去データ | Grok |
| **情報リサーチ** | オンライン情報 | Grok/Gemini |
| **エビデンスリサーチ** | オンライン情報 + 一次情報源 | PowerPlay等（確実な情報源重視） |

#### SNSカバレッジ
- X（Twitter）: Grokで強力にカバー
- Instagram: 有料ツール検討
- YouTube: 有料ツール検討
- TikTok: 有料ツール検討

#### UI構成
サイドバーに「リサーチ」配下に4つのサブ機能を配置：
```
リサーチ
├── 出演者リサーチ
├── 場所リサーチ
├── 情報リサーチ
└── エビデンスリサーチ
```

### 3.4 新企画立案

#### 入力
- 番組特性（過去の企画・特性を学習）
- 新企画の方向性・テーマ

#### 出力
- 新企画案（複数案提案）
- 企画タイトル
- コンセプト
- 内容構成

#### 注意点
- **番組データは他チームから見えないように厳重に管理**
- 過去の企画を参考にするが、AIに判断させすぎない
- ディレクター・プロデューサーが最終判断

## 4. 技術仕様

### 4.1 AIモデル選定

| 用途 | モデル | 理由 |
|------|--------|------|
| テキスト整形（議事録/NA） | Gemini 2.5 Flash Lite | 最安値、無料枠あり |
| リサーチ（X検索） | Grok | X検索に強い |
| エビデンスチェック | PowerPlay等 | 確実な情報源を重視 |
| 新企画立案 | Grok/Gemini | バランス型 |

### 4.2 出力形式

| 機能 | プライマリ出力 | 備考 |
|------|---------------|------|
| NA原稿作成 | Word (.docx) | 編集が必要なためWord必須 |
| 議事録作成 | Word (.docx) / Markdown | 編集可能な形式 |
| リサーチ | Markdown → Word | 最終的にWordで編集 |
| 新企画立案 | Markdown → Word | 同上 |

### 4.3 技術的制約

#### Word/PowerPoint出力の問題
- AIはWord/PowerPointレイアウトが苦手
- 画像を含むと崩れやすい
- **対応策**: 
  - テキスト主体のMarkdownで出力
  - ユーザーがWordにコピー&ペーストして最終調整
  - 構成資料作成は優先度を下げる

## 7. 非機能要件

### 7.1 セキュリティ
- Google認証（社内ドメイン制限）
- 番組データは他チームから隔離
- APIキーは環境変数で管理

### 7.2 パフォーマンス
- AI応答はストリーミング表示
- ファイルアップロードは最大10MB
- レスポンスタイム: 3秒以内（ファーストレスポンス）

### 7.3 コスト管理
- 無料枠を優先使用（Gemini無料枠）
- コストの高いモデルは必要な機能のみ使用
- 月次コストモニタリング
