# 人間の介入ポイント

> 本ドキュメントは [Agent Swarm 並列開発ガイド](./agent-swarm-guide.md) の一部を切り出したものです。

自律実行方針に従い、エージェントは基本的に**許可を取らずに進行**します。以下のタイミングでのみ**人間（Orchestrator/開発者）の判断・操作**が必要です。

## Wave 1 開始前（必須）✅ 済

| 項目 | 内容 | 担当 | 状態 |
|------|------|------|------|
| **APIキー取得** | Google AI Studio で Gemini APIキーを取得 | 開発者 | ✅ 済 |
| **APIキー取得** | xAI で Grok APIキーを取得（$25無料クレジット） | 開発者 | ✅ 済 |
| **APIキー取得** | Perplexity APIキーを取得（必要なら） | 開発者 | ✅ 済 |
| **環境変数設定** | `.env.local` に APIキーを設定 | 開発者 | ✅ 済 |
| **Google Cloud設定** | Google OAuth クライアントID/シークレット取得 | 開発者 | ✅ 済 |
| **リポジトリ準備** | GitHubリポジトリ作成、アクセス権設定 | 開発者 | ✅ 済 |
| **Upstash/Neon** | Redis（キャッシュ）・DB（Neon）の設定（必要なら） | 開発者 | 任意 |

## Wave 1 中（判断が必要な場合）✅ 済

| 項目 | 内容 | 判断基準 | 状態 |
|------|------|----------|------|
| **UIデザイン確定** | ui-research の成果を確認・承認 | デザインに問題があれば修正指示 | ✅ 済 |
| **DB設計レビュー** | database-schema の Prismaスキーマ確認 | モデル定義に問題があれば修正 | ✅ 済 |
| **認証設定** | Google Workspace SSO の動作確認 | ログインテストが必要 | ✅ 済 |

## Wave 2〜3 中（並列開発中）🔄 進行中

| 項目 | 内容 | 対応 |
|------|------|------|
| **コンフリクト解消** | 複数エージェントが同じファイルを編集 | 人間が優先順位を決定 |
| **仕様変更** | 開発中に要件が変わった場合 | 人間が判断してエージェントに指示 |
| **エラー対応** | 解決困難なエラーが発生 | 人間が原因調査・対策指示 |
| **LLM選定変更** | モデル切り替えの判断 | コスト/性能トレードオフの判断 |
| **エージェント遅延時** | ステップが長い・タイムアウト・ループ気味 | Perplexity MCP で調査、タスク簡略化や手動介入を検討 |

## Wave 3 後半開始時（判断）⏳ 予定

| 項目 | 内容 | 判断基準 |
|------|------|----------|
| **pj-d-schedule 開始** | pj-a/pj-b/pj-c の UI パターンが確定しているか確認 | 確定後に並列開始可能 |
| **google-drive 開始** | pj-c-research の Drive 連携要件が確定しているか確認 | 確定後に開始可能 |

## Wave 3 完了時（必須）⏳ 予定

| 項目 | 内容 | 担当 |
|------|------|------|
| **動作確認** | 全機能の動作テスト | 開発者/テスター |
| **承認** | 機能の仕様通りの動作を確認 | 開発者/ステークホルダー |
| **有料API判断** | 無料枠超過時の課金判断 | 開発者/管理者 |

## Wave 4 中（必須）⏳ 予定

| 項目 | 内容 | 担当 |
|------|------|------|
| **Vercelデプロイ** | 本番環境へのデプロイ実行 | 開発者 |
| **ドメイン設定** | カスタムドメインの設定 | 開発者 |
| **監視設定** | エラー監視、アラート設定 | 開発者 |

## 継続的な介入（運用時）

| 項目 | 内容 | 頻度 |
|------|------|------|
| **コスト監視** | 月額コストの確認 | 月1回 |
| **API使用率** | Google AI Studio / xAI / Perplexity 無料枠の確認 | 週1回 |
| **キャッシュ・Redis** | Upstash Redis 使用量・レート制限の確認（[cache-rate-limit.md](../logs/cache-rate-limit.md) 参照） | 週1回 |
| **アクセス権管理** | 新規ユーザー/退職者の対応 | 随時 |
| **バグ報告対応** | ユーザーからの不具合報告 | 随時 |
