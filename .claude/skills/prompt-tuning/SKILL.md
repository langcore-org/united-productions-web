# プロンプトチューニングスキル

> **スキル名**: prompt-tuning  
> **用途**: AIプロンプトの効果的な改善・最適化  
> **対象**: 出演者リサーチ、エビデンスリサーチ、その他LLM機能

---

## 概要

このスキルは、United Productions様向けAI Hub（Teddy）のプロンプト品質を向上させるための体系的アプローチを提供する。

## チューニングプロセス

### Phase 1: 現状分析（Day 1）

```markdown
1. 現在のプロンプトを取得
   - lib/prompts/ 配下の該当ファイルを確認
   - システムプロンプトとユーザープロンプトを分離

2. 出力サンプル収集（10〜20件）
   - 実際の検索クエリでテスト
   - 成功例・失敗例を分類

3. 問題点の特定
   - 情報の欠落
   - 形式の不統一
   - 誤情報の混入
   - 応答速度
```

### Phase 2: 改善案設計（Day 2）

```markdown
1. 出力形式の明確化
   - マークダウン構造の定義
   - 必須項目の洗い出し
   - 任意項目の区別

2. 検索戦略の最適化
   - ツール使い分け（Web/X検索）
   - 検索順序の最適化
   - 情報源の信頼性評価基準

3. プロンプト構造の再設計
   - システムプロンプトの分割
     - 役割定義
     - 出力形式指示
     - 検索戦略
     - 注意事項
```

### Phase 3: 反復テスト（Day 3-4）

```markdown
1. A/Bテスト実施
   - 旧プロンプト vs 新プロンプト
   - 同じクエリで比較

2. 評価基準での採点
   | 項目 | 配点 | 目標 |
   |------|------|------|
   | 情報網羅性 | 30 | 80%以上 |
   | 正確性 | 30 | 誤情報なし |
   | 読みやすさ | 20 | そのまま使える |
   | 応答速度 | 20 | 30秒以内 |

3. 改善サイクル
   - 問題箇所の特定
   - プロンプト修正
   - 再テスト（3回まで）
```

### Phase 4: 確定・展開（Day 5）

```markdown
1. 最終プロンプトの確定
2. バージョン管理（lib/prompts/配下に保存）
3. 本番適用
4. 効果測定の仕組み設置
```

## プロンプトテンプレート構造

### システムプロンプト基本構造

```markdown
# 役割
{AIの役割を簡潔に}

# 検索戦略
1. {優先する情報源}
2. {検索ツールの使い分け}
3. {情報の統合方法}

# 出力形式
```
{厳密なフォーマット指定}
```

# 注意事項
- {重要な制約1}
- {重要な制約2}
- {除外すべき情報}
```

### Few-Shot例の活用

```markdown
## 良好な出力例

入力: {クエリ例}
出力:
```
{理想的な出力例}
```

## 避けるべき出力例

入力: {クエリ例}
誤った出力:
```
{問題のある出力例}
```
理由: {なぜ問題か}
```

## よくある失敗と対策

### 失敗1: 思考過程の出力

**症状**: AIの思考过程がユーザーに表示される

**対策**:
```markdown
システムプロンプトに追加:
「思考過程は出力に含めないでください。最終結果のみを出力してください。」
```

### 失敗2: 出力形式のばらつき

**症状**: 同じクエリでも形式が変わる

**対策**:
```markdown
厳密なフォーマットを指定:
"以下の厳密な形式で出力してください。項目がない場合は「なし」と記載。"
```

### 失敗3: 古い情報の混入

**症状**: 過去の情報が最新として表示される

**対策**:
```markdown
時期確認を指示:
"各情報について、取得年月または情報の年月を必ず明記してください。"
```
