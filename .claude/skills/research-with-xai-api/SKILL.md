# Research with xAI API

xAI Grok APIのサーバーサイドツール（web_search, x_search）を活用して、最新情報・口コミ・技術動向を効率的にリサーチするスキル。

## いつ使うか

- 最新の技術情報、ライブラリのバージョン、APIの仕様変更を調べたい時
- SNS（X）上の口コミ、評判、ユーザーの声を収集したい時
- 複数ソースを横断した包括的なリサーチが必要な時
- ファクトチェックや情報の裏取りをしたい時

## ベストプラクティス

### モデル選択

| モデル | 用途 | コスト目安 |
|--------|------|-----------|
| `grok-4-1-fast-reasoning` | 高速リサーチ、簡潔な回答 | ~$0.018/リクエスト |
| `grok-4.20-multi-agent-beta` | 深い調査、複数ツール連携 | ~$0.30/リクエスト |

- **まず 4.1 Fast で概要を掴み、深掘りが必要なら 4.2 Multi-Agent を使う**
- 両モデルの出力を比較して、より質の高い方を採用するのも有効
- サーバーサイドツール呼び出しコスト: $5.00/1,000回（モデル共通）

### サーバーサイドツール

```
web_search  - 一般的なWeb検索。技術ドキュメント、ニュース、公式情報の取得
x_search    - X（Twitter）検索。リアルタイムの口コミ、トレンド、ユーザー反応
```

- ツールはGrok APIが自動的に判断して呼び出す（明示的指定不要）
- プロンプトに「最新の」「Xでの評判」等を含めると適切なツールが選ばれやすい
- 4.2 Multi-Agentは内部で複数回ツールを呼ぶため、1リクエストで深い調査が可能

### プロンプト設計のコツ

- **具体的な質問**にする（「〇〇について教えて」より「〇〇の2026年3月時点での最新バージョンと主な変更点」）
- **検索意図を明示**する（「Xでのユーザーの評判を調べて」「公式ドキュメントから確認して」）
- **期間を指定**する（「直近1週間」「2026年以降」）
- citations（引用）が返るので、ソースの信頼性を確認する

## ワークフロー例

以下は固定的な手順ではなく、状況に応じて組み合わせる調査パターン。

### パターン1: 広く浅く → 深く狭く

1. 4.1 Fast で概要を素早く把握（web_search）
2. 気になるトピックを 4.2 Multi-Agent で深掘り
3. X検索で実際のユーザー体験・評判を補完

### パターン2: 比較検証

1. 同じ質問を両モデルに投げる
2. 出力を比較し、一致する情報は信頼度が高いと判断
3. 食い違う点は追加検索で裏取り

### パターン3: リアルタイム情報収集

1. x_search で最新の話題・反応を収集
2. web_search で公式情報・詳細を確認
3. 時系列で整理して傾向を把握

### パターン4: 技術調査

1. web_search で公式ドキュメント・GitHub Issues を確認
2. x_search で開発者コミュニティの実体験を収集
3. 既知の問題・ワークアラウンドを整理

## コスト意識

- 社内にRAGがある場合は**まずRAGを検索し、不足分だけAPI検索**する（60-80%コスト削減の見込み）
- 4.2 Multi-Agent は1リクエストで内部的に多数のツール呼び出しを行うため、コストが高い
- 単純な事実確認なら 4.1 Fast で十分
- リサーチ結果は必ずドキュメントに残し、同じ調査の繰り返しを避ける

## API呼び出し例（Bash）

```bash
# grok-4-1-fast-reasoning でWeb検索付きリサーチ
curl -s https://api.x.ai/v1/responses \
  -H "Authorization: Bearer $XAI_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{
    "model": "grok-4-1-fast-reasoning",
    "tools": [{"type": "web_search"}, {"type": "x_search"}],
    "input": "質問内容をここに"
  }' | python3 -c "
import sys, json
data = json.load(sys.stdin)
for item in data.get('output', []):
    if item.get('type') == 'message':
        for c in item.get('content', []):
            if c.get('type') == 'output_text':
                print(c['text'])
"
```
