# TV番組データ管理スキル

Teddy（AIアシスタント）が参照する `lib/knowledge/` のTV番組データの管理・検証・更新を行う。

## クイックスタート

### 新規データ追加時

```
1. テンプレート生成
   node scripts/collect-from-official.mjs

2. 公式サイトでデータ収集
   → 最低2つの独立した情報源で確認

3. 多角的検証
   → 放送局公式サイト + 番組公式SNS + 見逃し配信等

4. データ投入
   → lib/knowledge/programs-*.ts
```

### 既存データ検証時

```
1. 検証スクリプト実行
   node scripts/verify-episodes.mjs

2. 問題発見時は即座に無効化
   recentEpisodes: []  // 一時的に空配列に

3. 正確なデータを再収集

4. 再検証後、有効化
```

## 基本原則

### 🔒 厳格な多角的検証

**必須**: 複数の独立した情報源で裏付けが取れて初めて正しい情報と認定する。

```
Tier 1（必須）:
├── 放送局公式サイト
├── 番組公式サイト
└── 番組公式SNS/X

Tier 2（補助・検証用）:
├── TVer/Abema等見逃し配信
├── Gガイド（テレビ番組表）
└── ニュース記事

Tier 3（参考のみ）:
└── Wikipedia
```

### ⚠️ データは嘘をつく

- **AI生成データ**は必ず検証する
- **曜日と日付**はカレンダーで照合する
- **存在しない放送回**が含まれていないか確認する
- **コミット前**にサンプル検証（10-20%）を実施する

## 緊急時対応フロー

誤りが見つかった場合：

```
1. 即座の無効化
   recentEpisodes: []  // コメントで理由を記載

2. 検証レポート作成
   docs/verification/report-YYYYMMDD.md

3. 原因調査と修正

4. 再検証

5. 問題なければ有効化
```

## 品質チェックリスト

```
□ 一次情報源（公式サイト）を確認したか？
□ 2つ以上のソースで裏付けを取ったか？
□ 日付と曜日が整合しているか？
□ AI生成データではないか？
□ 存在しない放送回が含まれていないか？
□ 関連ファイルの整合性は確認したか？
```
