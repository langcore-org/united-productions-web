# 多角的検証ガイド

TV番組データの厳格な多角的検証プロセスと情報源の詳細ガイド。

## 基本原則

**原則**: **複数の独立した情報源で裏付けが取れて初めて正しい情報と認定する**

```
✅ 正しい検証フロー:

【Step 1: 一次情報源収集】
  ├─ 放送局公式サイト（必須）
  ├─ 番組公式サイト（必須）
  ├─ 番組公式SNS/X（必須：最新情報）
  └─ 番組公式Instagram（補助）

【Step 2: 二次情報源確認】
  ├─ TVer/Abema等見逃し配信（タイトル確認）
  ├─ Gガイド/番組表サイト（時刻確認）
  └─ ニュース記事（ゲスト確認）

【Step 3: 相互検証】
  ├─ 2つ以上のソースで一致 → ✅ 正しいと認定
  ├─ 矛盾がある → 一次情報源を優先
  └─ 不明瞭 → 「未確認」としてマーク

【Step 4: 最終確認】
  └─ 収集した情報を元に、再度公式サイトで確認
```

## 検証情報源の優先順位

```
Tier 1（必須・最優先）:
├── 放送局公式サイト（ntv.co.jp, tbs.co.jp等）
├── 番組公式サイト（番組個別ページ）
├── 番組公式X/Twitter（@program_name）
└── 番組公式Instagram

Tier 2（補助・検証用）:
├── TVer/Abema等見逃し配信サイト
├── Gガイド（テレビ番組表）
└── 新聞/ニュースサイトの番組紹介記事

Tier 3（参考のみ）:
├── Wikipedia
└── ファンサイト/まとめブログ
```

## 情報源別の確認ポイント

### 放送局公式サイト

```
確認項目:
□ 番組トップページの「番組概要」
□ 「過去の放送」または「放送予定」ページ
□ 番組公式SNSへのリンク
□ お問い合わせ先（必要時）

注意:
- サイトリニューアルで過去データが消えることがある
- 直近1ヶ月分は確実に保持されていることが多い
```

### 番組公式SNS/X

```
確認項目:
□ 各回放送前の予告投稿（日付・タイトル・ゲスト）
□ 放送後の振り返り投稿
□ 固定ツイート/プロフィール情報
□ ハッシュタグ（#番組名）の検索

検索コマンド例:
"from:program_account 2026-01-13"  # 特定日付の投稿
"#番組名 2026年1月"               # 月別検索
```

### 見逃し配信サイト（TVer/Abema等）

```
確認項目:
□ 各回のタイトル表記
□ 放送日の記載
□ あらすじ/内容紹介
□ 出演者リスト

利点:
- 公式サイトより詳細なタイトルが記載されていることが多い
- 過去の放送を遡って確認できる
```

## 検証必須チェックリスト

### 基本情報検証

```
□ 放送局公式サイトで番組ページを確認
□ 番組公式サイトで詳細を確認  
□ 番組公式SNSで最新情報を確認
□ 番組表サイト（Gガイド等）で時間を確認
□ 2つ以上のソースで情報が一致することを確認
□ Wikipediaは補助的にのみ使用
```

### 放送回データ検証（厳格版）

```
□ 【一次情報源】公式サイトの「過去の放送」欄を確認
□ 【一次情報源】公式SNSで各回放送の投稿を確認
□ 【相互検証】日付と曜日が一致するかカレンダーで検証
□ 【相互検証】タイトルが複数ソースで一致するか
□ 【一次情報源】ゲストが公式サイト/SNSで確認できるか
□ 【整合性】存在しない放送回（未来の日付など）がないか
□ 【裏付け】少なくとも2つの独立した情報源で確認
```

## 相互検証マトリクス

情報を正しいと認定するための最低限のソース数：

| 情報タイプ | 必要ソース数 | 必須ソース |
|-----------|-------------|-----------|
| 放送局 | 2つ以上 | 公式サイト + 番組表 |
| 放送時間 | 2つ以上 | 公式サイト + 番組表 |
| タイトル | 2つ以上 | 公式サイト + SNS/見逃し配信 |
| ゲスト名 | 2つ以上 | 公式サイト + SNS |
| 放送日 | 2つ以上 | SNS + 見逃し配信/番組表 |

## 矛盾が見つかった場合の対応

```
状況: ソースAとソースBで情報が矛盾

対応フロー:
1. 一次情報源を特定
   - 放送局公式サイト > 番組表サイト
   - 公式SNS > ファンサイト

2. 時系列で考える
   - より新しい情報が正しい可能性が高い
   - ただし、リニューアルで過去データが失われた可能性も

3. 詳細度で判断
   - より詳細な情報の方が信頼性が高い
   - 例：「ゲスト出演」>「ゲストあり」

4. 最終手段
   - 判断できない場合は「未確認」としてマーク
   - ユーザーに確認する
```

## 検証記録テンプレート（厳格版）

```markdown
## 検証記録: [番組名] [放送日]

### 対象情報
- 放送日: YYYY-MM-DD
- タイトル: [タイトル]
- ゲスト: [ゲスト名]

### 情報源と確認結果

| 情報源 | URL | 確認項目 | 結果 | メモ |
|--------|-----|---------|------|------|
| 放送局公式サイト | [URL] | タイトル・ゲスト | ✅一致 | - |
| 公式X | [URL] | タイトル・ゲスト | ✅一致 | 放送前投稿 |
| TVer | [URL] | タイトル | ✅一致 | - |
| Gガイド | [URL] | 放送時間 | ✅一致 | - |

### 検証結論
- [ ] 全ソースで一致 → ✅ 正しいと認定
- [ ] 一部矛盾 → [一次情報源を優先]
- [ ] 不明瞭 → ⏸️ 未確認としてマーク

### 確認者・日時
- 確認者: [名前]
- 確認日時: YYYY-MM-DD HH:MM
```

## 実践例

### マツコの知らない世界（2026-01-13）

```
【検証対象】
放送日: 2026年1月13日（火）
タイトル: 「御守りの世界」
ゲスト: 林直岳

【情報源確認】

1. TBS公式サイト
   URL: https://www.tbs.co.jp/matsukosekai/
   結果: 「御守りの世界」を確認 ✅

2. 公式X (@matsukosekai)
   投稿: 2026-01-12 "明日よる8時は『御守りの世界』"
   結果: タイトル一致 ✅

3. TVer
   タイトル: "御守りの世界"
   ゲスト: 林直岳（御守りマニア）
   結果: タイトル・ゲスト一致 ✅

4. Gガイド
   放送時間: 火曜20:00-21:00
   結果: 時間一致 ✅

【結論】
4つの独立した情報源で確認済み → ✅ 正しいと認定
```

## 時系列データ特有の検証

| 項目 | 検証方法 | 重要度 |
|------|---------|--------|
| **日付と曜日** | カレンダー対照 | 🔴 必須 |
| **放送回の連続性** | 番号の欠落チェック | 🔴 必須 |
| **時間の重複** | スケジュール確認 | 🟡 重要 |
| **期間の整合性** | 開始日≦終了日 | 🔴 必須 |

## 検証スクリプト

### verify-episodes.mjs

xAI API（Grok）を使用した自動検証スクリプト。

```bash
node scripts/verify-episodes.mjs
```

**用途**: 複数番組の放送回データを一括検証  
**特徴**:
- Web検索ツールを自動有効化
- ストリーミングレスポンスでリアルタイム表示

**注意**:
- xAI API検証は**参考情報**として扱う
- 必ず一次情報源で最終確認
- 自動検証の「誤り」判定に踊らされない
