# 誤りパターン集

TV番組データでよく発生する誤りパターンと対策の詳細ガイド。

## 放送回データ特有の誤り

| パターン | 具体例 | 検出方法 | 影響 |
|---------|--------|---------|------|
| **放送局誤り** | 日本テレビ→フジテレビ、Eテレ→総合など | 公式サイト確認 | 🔴 高（見逃し） |
| **曜日・時間誤り** | 水曜24:15→23:15 | 番組表確認 | 🔴 高（見逃し） |
| **AI生成タイトル** | 架空のテーマ名 | 公式サイト確認 | 🟡 中（信頼性） |
| **日付誤り** | 存在しない日付 | カレンダー照合 | 🔴 高（検索不能） |
| **ゲスト誤り** | 存在しないゲスト名 | 複数ソース確認 | 🟡 中（信頼性） |
| **曜日不一致** | 「3/15（火）」→実際は金曜 | カレンダー照合 | 🔴 高（混乱） |
| **存在しない放送回** | 実際には放送されていない日付 | 公式サイト/SNS確認 | 🔴 高 |

## 原因別対策

| 原因 | 対策 |
|------|------|
| **AI生成データ** | 必ず公式サイトで検証 |
| **情報の陳腐化** | 改編期（4月・10月）に定期検証 |
| **過去の改編見逃し** | 四半期ごとに見直し |
| **複数ファイルの不整合** | 修正時に関連ファイルも確認 |
| **一次情報源未確認** | 複数の独立した情報源で裏付け |

## 実例集

### 誤り例1: AI生成タイトル

```
【誤り】
タイトル: "最新AI技術で変わるテレビ制作の世界"

【実際】
タイトル: "マンガ描き文字の世界"

【検出方法】
TBS公式サイトで「過去の放送」を確認したところ、
「マンガ描き文字の世界」と記載されていた。

【原因】
AIが文脈から推測した架空のタイトル

【対策】
必ず公式サイトの「過去の放送」欄を確認
```

### 誤り例2: 放送局誤り

```
【誤り】
放送局: 日本テレビ

【実際】
放送局: フジテレビ

【検出方法】
フジテレビ公式サイトで番組ページを発見。
日本テレビの番組表には該当番組がなかった。

【原因】
初期データ作成時の誤入力

【対策】
放送局公式サイトの番組ページを必ず確認
```

### 誤り例3: 曜日不一致

```
【誤り】
放送日: 2026年3月15日（火）

【実際】
放送日: 2026年3月15日（金）

【検出方法】
カレンダーで3月15日を確認したところ、実際は金曜日。

【原因】
日付と曜日の対応を確認していなかった

【対策】
放送回データ作成時は必ずカレンダーで曜日を照合
```

### 誤り例4: 存在しない放送回

```
【誤り】
放送日: 2026年1月2日
タイトル: "新春スペシャル"

【実際】
1月2日は放送なし（お正月休み）

【検出方法】
公式SNSで1月2日前後の投稿を確認したところ、
放送予告がなかった。

【原因】
AIが「新春=1月2日」と推測

【対策】
公式SNSで放送予告を確認
```

### 誤り例5: xAI検証結果と実際の乖離

```
【状況】
xAI検証: 「2026-01-02 藤田ニコル新婚SPは不存在」

【実際】
「お笑い研究部 エバース前半戦」が正しく放送されていた

【原因】
Web検索で情報が見つからなかったが、
公式サイトには正しく記載されていた

【対策】
xAI検証は参考情報として扱い、
必ず一次情報源（公式サイト）で最終確認
```

## 予防策

### データ作成時

```
□ 公式サイトのURLを最初に確認
□ 複数の情報源で裏付けを取る
□ 日付と曜日の整合性をカレンダーで確認
□ AI生成の可能性があるデータに注意
```

### コミット前

```
□ サンプル検証（10-20%）を実施
□ 放送局・時間の整合性を確認
□ 曜日と日付の整合性を確認
□ 存在しない日付が含まれていないか確認
```

### 定期検証時

```
□ 改編期（4月・10月）は全番組を確認
□ 時間変更がないか確認
□ 新番組・終了番組をチェック
□ 直近の放送回データをサンプル検証
```

## 緊急時対応

### 誤り発見時の対応フロー

```
1. 即座の無効化
   recentEpisodes: []  // 一時的に空配列に
   
   コメント例:
   // 誤りが見つかったため一時的に無効化（2026-02-27）
   // 詳細: docs/verification/report-20260227.md

2. 検証レポート作成
   docs/verification/report-YYYYMMDD.md
   
   内容:
   - 発見された誤りの詳細
   - 影響範囲
   - 原因分析
   - 修正予定

3. 原因調査
   - なぜ誤りが混入したか
   - 検証プロセスの抜け穴を特定

4. 修正と再検証
   - 正確なデータを収集
   - 多角的検証を実施
   - 問題なければ有効化

5. 再発防止
   - 検証プロセスの改善
   - チェックリストの更新
```
